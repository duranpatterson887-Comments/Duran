CoinMarketCap authenticates API requests using a unique API key tied to your account and plan. Every authenticated call must include this key so the platform can identify your account, track credit usage, and enforce rate limits. The recommended method is to pass the key as an HTTP request header — keeping it out of URLs, browser history, and server logs. For quick evaluation without an account, the Keyless trial endpoint lets you hit a subset of endpoints with no credentials at all.

## Getting Your API Key

<Steps>
  <Step title="Create a free account">
    Visit [pro.coinmarketcap.com/signup](https://pro.coinmarketcap.com/signup) and register with your email address. No credit card is required for the free Basic plan, which includes 15,000 credits per month.
  </Step>
  <Step title="Confirm your email">
    Check your inbox for a verification email from CoinMarketCap and click the confirmation link to activate your account.
  </Step>
  <Step title="Retrieve your key from the dashboard">
    Log in to [pro.coinmarketcap.com](https://pro.coinmarketcap.com), navigate to **Dashboard → API Keys**, and copy your key. It is a UUID string in the format `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`.
  </Step>
</Steps>

<Warning>
  Your API key grants full access to your account's credits and data entitlements. Treat it like a password — never commit it to source control or expose it in client-side code. If a key is compromised, regenerate it immediately from the dashboard.
</Warning>

## Using Your API Key

### Header Method (Recommended)

Pass your key in the `X-CMC_PRO_API_KEY` request header. This approach keeps the key out of access logs and browser history.

<CodeGroup>
  ```bash cURL
  curl -H "X-CMC_PRO_API_KEY: YOUR_API_KEY" \
    "https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest?symbol=BTC"
  ```

  ```python Python
  import requests

  headers = {'X-CMC_PRO_API_KEY': 'YOUR_API_KEY'}
  params = {'symbol': 'BTC', 'convert': 'USD'}

  response = requests.get(
      'https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest',
      headers=headers,
      params=params
  )
  print(response.json())
  ```

  ```javascript Node.js
  const axios = require('axios');

  const response = await axios.get(
    'https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest',
    {
      headers: { 'X-CMC_PRO_API_KEY': 'YOUR_API_KEY' },
      params: { symbol: 'BTC', convert: 'USD' },
    }
  );
  console.log(response.data);
  ```
</CodeGroup>

### Query Parameter Method (Alternative)

You can also append your key as a query parameter. Use this method only in server-to-server contexts where you control the full request lifecycle.

```bash
curl "https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest?symbol=BTC&CMC_PRO_API_KEY=YOUR_API_KEY"
```

<Note>
  The query parameter method is supported for convenience, but the header method is strongly preferred. Query parameters can appear in server access logs and browser history, increasing the risk of accidental exposure.
</Note>

## Keyless Access

If you want to explore the API before creating an account, use the **Keyless trial endpoint**. No API key, no sign-up — just make a request and get data back.

**Trial base URL:**
```
https://pro-api.coinmarketcap.com/trial-pro-api
```

```bash
# No API key header needed
curl "https://pro-api.coinmarketcap.com/trial-pro-api/v2/cryptocurrency/quotes/latest?symbol=BTC"
```

Keyless access has the following constraints:

| Constraint | Limit |
|---|---|
| Rate limit | 10 requests per minute |
| Endpoints available | Limited subset (quotes, listings, global metrics) |
| Historical data | None |
| WebSocket streaming | Not available |
| Credit tracking | Not available |

<Tip>
  Keyless access is ideal for proof-of-concept work and AI-agent pipelines. For production integrations, the free **Basic plan** at [pro.coinmarketcap.com/signup](https://pro.coinmarketcap.com/signup) gives you 15,000 tracked credits per month with a full dashboard.
</Tip>

### x402 Pay-Per-Request

For serverless and AI-agent workflows that need authenticated access without managing API keys, CoinMarketCap supports the **x402 protocol**. Pay $0.01 USDC on-chain per request — no account or key required. See the [x402 Keyless API guide](/guides/x402-keyless) for integration details.

## Security Best Practices

<Warning>
  **Never embed your API key in frontend JavaScript, mobile app binaries, or public repositories.** Keys exposed in client-side code can be extracted by anyone who inspects your bundle or network traffic.
</Warning>

Follow these practices to keep your key secure:

- **Use environment variables.** Load the key at runtime from `process.env.CMC_API_KEY` (Node.js) or `os.environ['CMC_API_KEY']` (Python) rather than hardcoding it.
- **Use a backend proxy.** Route all CoinMarketCap API calls through your own server so the key never reaches the browser.
- **Rotate keys regularly.** Regenerate your key from the dashboard on a schedule, or immediately if you suspect exposure.
- **Restrict usage in the dashboard.** Where available, bind your key to specific IP ranges or referrer domains to limit blast radius if it leaks.

```bash
# Good — load from environment
curl -H "X-CMC_PRO_API_KEY: $CMC_API_KEY" \
  "https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest?symbol=BTC"
```

```python
import os, requests

headers = {'X-CMC_PRO_API_KEY': os.environ['CMC_API_KEY']}
response = requests.get(
    'https://pro-api.coinmarketcap.com/v2/cryptocurrency/quotes/latest',
    headers=headers,
    params={'symbol': 'BTC'}
)
```

## Authentication Error Codes

When a request fails due to an authentication problem, the `status` object in the response body includes an `error_code` and a human-readable `error_message`.

| Error Code | Meaning | Resolution |
|---|---|---|
| `1001` | Invalid API key — the key is not recognised | Check for typos; copy the key fresh from your dashboard |
| `1002` | API key missing — no key was included in the request | Add the `X-CMC_PRO_API_KEY` header or `CMC_PRO_API_KEY` query param |
| `1006` | Subscription plan insufficient — the endpoint requires a higher-tier plan | Review your current plan and upgrade at [pro.coinmarketcap.com](https://pro.coinmarketcap.com) |

```json
{
  "status": {
    "timestamp": "2025-01-15T12:00:00.000Z",
    "error_code": 1002,
    "error_message": "API key missing.",
    "elapsed": 2,
    "credit_count": 0
  }
}
```

<Note>
  A `credit_count` of `0` in the response means no credits were deducted for the failed request. You are never charged for authentication errors.
</Note>
